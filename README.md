# 📁 PM Templates — Mohsin Ali

A collection of practical project management templates — built from real experience managing software delivery as a QA & Automation Engineer turned Associate Project Manager.

These aren't textbook templates. They're structured around what actually matters in software project delivery.

---

## 📋 Templates Included

| Template | Description | Best For |
|---|---|---|
| [📄 Project Charter](./project-charter-template.md) | Full project definition — scope, goals, stakeholders, budget, risks | Kickstarting any new project |
| [⚠️ Risk Register](./risk-register-template.md) | Identify, score, and track project risks | Ongoing risk management |
| [👥 RACI Matrix](./raci-matrix-template.md) | Define who does what across every task | Avoiding confusion on large teams |
| [🏃 Sprint Planning](./sprint-planning-template.md) | Full sprint planning + retrospective format | Agile / Scrum teams |

---

## 🧠 How I Use These

Coming from a QA background, I've always cared about **what can go wrong before it does**. These templates reflect that:

- The **Risk Register** is the first document I open on any project
- The **Project Charter** forces scope clarity before a single line of work begins
- The **RACI Matrix** eliminates the "I thought you were doing that" problem
- The **Sprint Planning** template includes a Definition of Done — because "done" should never be ambiguous

---

## 💡 How to Use

1. Click on any template above
2. Copy the raw markdown
3. Paste into your Notion, Confluence, Google Docs, or GitHub repo
4. Fill in your project details
